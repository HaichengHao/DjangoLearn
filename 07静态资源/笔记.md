## 静态资源  
- 开发需要:css,js,图片
> 放在根目录 /static/
> app目录下的 /static/文件夹下


- 媒体文件:用户上传的数据  
> 根目录的 /media/


### 静态文件  
在settings.py中这么写
```python

STATIC_URL = '/static/'
STATICFILES_DIRS = [
    BASE_DIR/'static',
]
```
这样的话它就会优先来这个文件里头找

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>测试首页</title>
</head>
<body>
    <img src="/static/monky.png" alt="猴哥" title="孙悟空">
</body>
</html>
```

它先从项目根目录找monky.png如果项目目录中没有,那就去app里头的static中找
其实一般开发的过程中禁止写成`/static/monky.png`,因为我们在主路由已经设置了STATIC_URL = '/static/'  
那么我们就可以利用{% load static %} 来将其加载进来  

修改
```html
{% load static %}
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>测试首页</title>
</head>
<body>
    <img src="user/monky.png" alt="猴哥" title="孙悟空">
</body>
</html>
```

这样其实相当于直接访问的是/static/user/monky.png


### 媒体文件  
一般放在media中,但是需要注意的是Django默认不直接处理，
所以我们需要进行的一步操作就是在主路由的urls上添加一些操作

```python
from django.contrib import admin
from django.urls import path,include
from django.conf.urls.static import static
from django.conf import settings
urlpatterns = [
    path('user/',include('apps.user.urls',namespace='user'))
] + static(settings.MEDIA_URL,document_root=settings.MEDIA_ROOT)

```


### 补充,如何高效的访问静态文件
```python
#settings.py中必须将INSTALLED_APPS设置'django.contrib.staticfiles'
INSTALLED_APPS = [
    # 'django.contrib.admin',
    # 'django.contrib.auth',
    # 'django.contrib.contenttypes',
    # 'django.contrib.sessions',
    # 'django.contrib.messages',
    'django.contrib.staticfiles',
    'apps.web',
    'apps.api'
]
STATIC_URL = '/static/'

#文件也一定要配置好
STATICFILES_DIRS = [
    BASE_DIR / 'static',  # 项目根目录下的 static 文件夹
]

```

最后一定要load将其加载进来
```html
{% load static %} 

<img src="{% static '文件路径' %}" alt="">
```
